# Доступность (a11y)

Цель: интерфейсом можно пользоваться с клавиатуры, скринридером и при разных возможностях.

## Чек-лист
- **Клавиатура**: всё достижимо и управляемо с клавиатуры; видимый фокус; логичный порядок табуляции; нет ловушек фокуса; Esc закрывает модалки.
- **Семантика/ARIA**: нативные элементы (button/a/label) вместо div с onClick; ARIA только когда нужно и корректно; роли/состояния (aria-expanded, aria-current) актуальны.
- **Формы**: каждый input связан с `<label>`; ошибки объявляются (aria-describedby/role=alert); подсказки доступны.
- **Контраст и текст**: контраст ≥ 4.5:1 для текста; не передавать смысл только цветом; масштабируемый текст (rem), не ломается при зуме 200%.
- **Изображения/медиа**: осмысленный `alt` (или пустой для декоративных); подписи/транскрипты для видео.
- **Структура**: один h1, иерархия заголовков, landmark-регионы (nav/main/footer), «skip to content».
- **Динамика**: изменения контента объявляются (aria-live) где важно; уважать prefers-reduced-motion.
- **Тач/таргеты**: цели ≥44px, достаточные отступы.

## Итог
Отметить нарушения доступности в затронутом UI и включить исправления в задачу.
