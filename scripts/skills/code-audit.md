# Аудит кода

Цель: системно найти проблемы и тех долг в затронутом коде и включить их в постановку («Попутно исправить»).

## Чек-лист
- **Дубли и мёртвый код**: повторяющаяся логика → вынести; неиспользуемые функции/экспорты/файлы/зависимости → удалить.
- **Обработка ошибок**: каждый await/IO обёрнут? Глотаются ли ошибки молча? Есть ли понятные сообщения? Нет ли `catch {}` без логирования там, где важно.
- **Типобезопасность**: `any`, небезопасные касты, отсутствующие проверки null/undefined, рассинхрон типов и реальных данных.
- **Граничные случаи**: пустые списки, 0/отрицательные, длинные строки, отсутствующие поля, гонки, повторный вызов (идемпотентность).
- **Состояние и побочные эффекты**: мутации общих структур, утечки (таймеры, подписки, соединения), эффекты в рендере.
- **Согласованность с конвенциями проекта**: именование, структура, паттерны — как в окружающем коде, не «по-своему».
- **Производительность по месту**: N+1 запросы, лишние перерисовки/циклы, тяжёлые операции в горячем пути.
- **Безопасность по месту**: валидация входных данных, экранирование, проверка прав, отсутствие секретов в коде.
- **Тесты/проверяемость**: покрыты ли критичные ветки; легко ли проверить результат.
- **Lint/типы**: ненулевой lint/tsc → довести до 0 (безопасные фиксы), рискованные правила — с обоснованием.

## Итог
Найденные дефекты — конкретно (что, где: файл/функция, как чинить) — добавить в описание задачи отдельным блоком, чтобы не копился тех долг.
