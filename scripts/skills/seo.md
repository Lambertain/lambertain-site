# SEO-оптимизация

Цель: страница хорошо индексируется и корректно представлена в поиске/соцсетях.

## Чек-лист
- **Мета**: уникальные `<title>` и `meta description` на каждой странице; длина в норме; ключевой смысл в начале.
- **Семантика**: один `<h1>` на страницу, корректная иерархия заголовков, семантические теги, `alt` у изображений.
- **Canonical и дубли**: `<link rel=canonical>`; нет дублей по параметрам/слешам; пагинация корректна.
- **Индексация**: `robots.txt`, `sitemap.xml` актуальны; нет случайного `noindex` на важных страницах; важный контент в HTML (SSR/SSG), а не только в JS.
- **Структурированные данные**: schema.org (Product/Article/Breadcrumb/Organization) где уместно; валидны.
- **OpenGraph/Twitter**: og:title/description/image, корректные превью для шаринга.
- **URL**: человекочитаемые, стабильные; редиректы 301 при переездах (не 302/цепочки).
- **Языки**: `hreflang` для мультиязычных; `lang` на `<html>`.
- **Скорость и мобильность**: Core Web Vitals и mobile-friendly влияют на ранжирование (см. скил performance).
- **Внутренняя перелинковка**: ключевые страницы доступны из навигации/ссылок.

## Итог
Перечислить недостающие/некорректные SEO-элементы для затронутых страниц как часть задачи.
