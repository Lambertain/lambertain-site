# Путь клиента (customer journey / UX-флоу)

Цель: пройти сценарий глазами пользователя и убрать трение, ничего не упустив.

## Чек-лист
- **Вход и онбординг**: первое впечатление, что делать с пустым состоянием, понятен ли следующий шаг без объяснений.
- **Ключевые сценарии**: пройти основной флоу по шагам (напр. каталог → товар → корзина → оплата → успех). На каждом шаге: ясно ли, что дальше? сколько кликов? нет ли тупиков.
- **Состояния**: загрузка (скелетоны/индикаторы), пусто, ошибка (понятное сообщение + что делать), успех (подтверждение). Не оставлять «мёртвых» экранов.
- **Формы**: минимум полей, понятные подписи, валидация по месту, сохранение введённого при ошибке, автофокус, удобство на мобиле.
- **Мобильный опыт**: тач-цели ≥44px, ничего не обрезается, клавиатура не ломает верстку, скролл/жесты работают.
- **Обратная связь**: каждое действие даёт отклик (тост/смена состояния), необратимые — с подтверждением.
- **Доступность пути**: можно отменить/вернуться назад; прогресс виден в многошаговых флоу.
- **Тексты**: на языке пользователя, без жаргона; ошибки — по-человечески.
- **Конверсия**: где пользователь может «отвалиться»? Снизить трение в этих точках.

## Итог
Описать целевой сценарий по шагам и точки трения, которые задача должна устранить.
