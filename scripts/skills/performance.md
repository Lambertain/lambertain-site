# Скорость загрузки и производительность

Цель: быстрый отклик и хорошие Core Web Vitals.

## Чек-лист
- **Core Web Vitals**: LCP (быстрый главный контент), INP/отзывчивость (не блокировать поток), CLS (нет скачков верстки — резервировать место под медиа/шрифты).
- **JS-бандл**: code splitting, динамический импорт тяжёлого, дерево зависимостей без лишнего, tree-shaking; не тащить большие либы ради мелочи.
- **Рендер-стратегия**: что можно SSR/SSG/стримить; не гидрировать лишнее; серверные компоненты где возможно.
- **Изображения**: современные форматы (webp/avif), нужные размеры, lazy-loading, width/height против CLS, оптимизация ассетов.
- **Шрифты**: `font-display: swap`, предзагрузка ключевых, сабсеттинг.
- **Сеть**: кэш-заголовки, CDN, сжатие (gzip/br), HTTP/2+, минимизация водопадов запросов, prefetch критичного.
- **Данные/БД**: устранить N+1, индексы под запросы, пагинация, не тянуть лишние поля, кэширование дорогих вычислений.
- **Клиент**: дебаунс/тротлинг, виртуализация длинных списков, мемоизация дорогих вычислений, отказ от лишних перерисовок.
- **Измерять**: до/после — Lighthouse/реальные метрики, а не на глаз.

## Итог
